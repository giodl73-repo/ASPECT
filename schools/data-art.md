# Data Art

## Origin and Claim

Data art emerges when data becomes the raw material for aesthetic expression rather than the subject of informational communication. Where statistical graphics asks "what do the numbers say?" and advocacy visualization asks "what should the audience do?", data art asks "what can the data become?" The form is aesthetic-first; the data provides structure, constraint, and material — not a message to be communicated.

Key practitioners: Jer Thorp (data and Society; flight patterns), Aaron Koblin (flight patterns, sheep market), Nathalie Miebach (weather data sculptures), Refik Anadol (machine hallucinations, AI data paintings), Nicholas Feltron (annual reports), Giorgia Lupi and Stefanie Posavec (*Dear Data*).

## Visual Grammar

Data art has no single grammar — it is the school defined by its departure from the grammars of other schools. What it shares:

### Constitutive Principles
1. **Data as material**: the dataset constrains and shapes the artwork; the data is not translated into abstract marks, it is made visible as itself
2. **Aesthetic primacy**: the visual experience is primary; information extraction is secondary or incidental
3. **Form follows feeling**: the visual form is chosen for its aesthetic and emotional properties first, then fitted to the data
4. **Legibility is negotiable**: the viewer may not be able to extract precise values; if the overall pattern or texture communicates something true about the data, this is sufficient

### Common Techniques
- **Mapping data to non-standard visual variables**: sound frequencies → line frequency; weather → sculpture material; flight paths → light trails
- **Aggregate texture**: large datasets rendered as texture, pattern, or field rather than as individual readable marks
- **Temporal accumulation**: data collected over time accumulates into a visual residue (Feltron's annual reports; Koblin's flight patterns overnight)
- **Cross-sensory encoding**: data encoded in media other than visual display (Miebach's knitted weather data)
- **Self-portrait data**: personal or behavioral data made visible (Dear Data)

## How the DEGAS Rubric Applies

Data art creates significant challenges for the rubric as currently structured:

- **School**: no single grammar to be faithful to; faithfulness is to the aesthetic intention and to the data's own structure
- **Truth**: accuracy is partially redefined — the aggregate texture should be true even if individual values are unreadable; encoding distortions for aesthetic effect are more permissible than in statistical graphics
- **Precision**: the concept inverts — richness and density may be deliberate; "economy" is not the correct criterion; better: does every visual choice serve the aesthetic/data relationship?
- **Effect**: Experiential type is almost always the primary type; Experiential (emergent-data) is frequently present simultaneously

## Status in the project
No canonical works reviewed. Candidates for first review: Jer Thorp's "Just Landed" (geographic Twitter patterns), Nathalie Miebach's hurricane data sculptures, or Refik Anadol's "Machine Hallucinations." Each would test a different variant of the school's grammar.

## Adjacent Schools
- **Abstract art**: both are aesthetic-primary; abstract art uses formal visual elements without data; data art uses data as the material
- **Statistical graphics**: data art uses the same underlying datasets but with aesthetic rather than informational priority
- **Scientific illustration**: both render; scientific illustration renders organisms for taxonomic accuracy; data art renders data for aesthetic experience
