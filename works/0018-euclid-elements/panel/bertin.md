# Panel Review — Bertin on 0018: Euclid's Elements

## Opening

I analyze visual marks and what they encode. Let me analyze what Euclid's marks encode, and then ask whether my framework is adequate to the task.

---

## Variable Analysis for Demonstrative Encoding

The standard Bertin visual variables (position, size, shape, value, hue, orientation, texture) describe how marks encode data properties. I will map them to the Euclidean diagram grammar:

| Variable | Use in Euclidean diagrams | Data type encoded |
|----------|--------------------------|-------------------|
| Position | Determines geometric relationship (intersection, adjacency, equidistance) | Nominal-relational — "this point is where these two arcs intersect" |
| Shape | Line vs. arc vs. point — distinguishes construction element types | Nominal — "this is a straight line"; "this is an arc" |
| Size | Length of line segments — NOT quantitative; longer does not mean "more" in any measured sense | Structural only — marks the existence of an element, not its quantity |
| Value | Not used | — |
| Hue | Not used | — |
| Orientation | Angle of line segments — encodes the direction of a line relative to others | Relational — "this line is perpendicular to that one" |
| Texture | Not used | — |

**Key finding:** Euclidean diagrams use only position, shape, and orientation from my framework — and they use them in a non-standard way. These variables do not encode data properties of a subject matter; they encode geometric relationships within a constructed argument.

Size is the most interesting case. A line drawn longer than another line in a Euclidean diagram does not claim to be longer in any measured sense. It claims only to be a different line element. The visual variable "size" is stripped of its quantitative and ordinal properties — it functions only nominally (to distinguish one line from another). This is a systematic re-assignment of variable properties that my grammar framework was not designed to accommodate.

This is why demonstrative encoding requires a rubric amendment: the marks have different encoding contracts than data-reporting marks.

## On Cluster I

Kandinsky has argued that demonstrative encoding is confirmed across works 0008 and 0018. I agree. My own framework is insufficient to analyze Euclidean diagrams as data visualizations, because they are not data visualizations — they are argument visualizations. The variables work differently; the encoding contract is different; the validity criterion is different.

**Cluster I amendment threshold confirmed: works 0008 + 0018 exhibit demonstrative encoding.**

I add a precision: demonstrative encoding has a variable grammar that is a strict subset of data-reporting encoding. It uses fewer variable properties (position: relational only; size: nominal only; orientation: relational only). The marks work by showing that a relationship holds, not by showing how much of something exists. This is a systematically restricted encoding mode.

## Innovation Flagged

**Innovation: Variable property reassignment in demonstrative encoding** — Rubric anchor: School Fidelity / Data/Story Integrity. In demonstrative encoding, standard visual variables (position, size) are used with systematically reassigned perceptual properties. Size becomes nominal (distinguishing elements) rather than quantitative or ordinal. Position becomes relational (indicating geometric relationship) rather than quantitative. This is not a grammar error — it is a systematically different encoding contract. The rubric should recognize that the same visual variable can operate under different perceptual contracts in different encoding modes.
