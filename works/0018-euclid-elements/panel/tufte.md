# Panel Review — Tufte on 0018: Euclid's Elements

## Opening

The first thing I want to say about Euclid's diagrams is this: they achieve the highest data-ink ratio of any work in this project. Every mark is required. There is no legend, no border, no title block, no gridline. There are lines, arcs, points, and letters. That is all.

The second thing I want to say is that "data-ink ratio" is the wrong measure for this work, and I will explain why.

---

## On Data-Ink Ratio and Demonstrative Encoding

My data-ink ratio principle assumes that there is a data set that the visualization is representing, and that the ratio of ink-devoted-to-data to total-ink should be maximized. The principle is: every mark not representing data is a candidate for removal.

In a Euclidean proof diagram, there is no data set. The marks do not represent quantities from a table. The marks constitute a procedure that, when followed, establishes a geometric claim. Every mark IS the argument.

I am therefore in an unusual position: my primary evaluative criterion (data-ink ratio) applies in spirit — nothing should be present that does not earn its place — but the thing that earns each mark's place is not data but logical necessity. A construction arc in I.1 is not a data mark but it is a necessary mark: without it, the proof does not work. Remove the arc and the argument fails.

The principle generalizes: **every mark in a valid proof diagram is logically necessary.** A mark that is not logically necessary is a proof error, not chartjunk. This is a stronger claim than my data-ink ratio: the standard data chart might tolerate slight redundancy without error; the proof diagram tolerates no redundancy, because redundancy in a proof means either the argument is repetitive (a minor logical failure) or the extra element is distractingly unnecessary (a presentational failure).

This is the Cluster I observation from my standpoint. I confirm: the diagram IS the argument, not a representation of the argument. The epistemic relationship is different. The rubric must name this.

**Cluster I amendment threshold confirmed: works 0008 + 0018. Demonstrative encoding requires rubric amendment to Data/Story Integrity.**

## 2. Data/Story Integrity /20 — Score: 15

I apply the demonstrative encoding re-interpretation, as Kandinsky and Bertin have argued:

- Logical foundation: sound for I.1-I.5. **No adjustment.**
- Demonstration completeness: complete. **No adjustment.**
- Logical validity: I.1 has the diagram-dependence problem (implicit circle intersection assumption). I.4 (the SAS congruence proposition) has a more significant issue: Euclid's proof uses "superposition" — placing one triangle on another to show they coincide — but this is not a valid Euclidean operation. Superposition assumes rigid motions that are not established by the postulates. **-2 adjustment.**
- Diagrammatic framework: correctly applies Euclidean plane geometry. **No adjustment.**

I.4's superposition issue is a real logical gap that later mathematicians have noted (Hilbert, among others). It is not a diagrammatic error per se — the diagram correctly depicts two congruent triangles — but it reflects a proof that relies on an implicit assumption (rigid motion preserves shape) that is not established in the system.

**Base: 17. Adjusted: -2 (I.4 superposition). Score: 15/20.**

## 3. Legibility /15 — Score: 11

The legibility for an accompanied student reading the primary Greek mathematical text is adequate. The legibility for anyone else is poor. The key barrier: no visual distinction between construction elements (tools) and figure elements (the result). A contemporary reader cannot identify, without instruction, which marks are "what we're constructing" and which marks are "what we used to construct it."

This is a structural legibility problem. Kandinsky would call it a grammar gap. I call it a missed economy opportunity: if the construction arcs were visually distinguished (lighter, dashed) from the final figure elements (heavier, solid), the legibility would be dramatically improved at minimal cost.

**Score: 11/15** [contemporary: 7/15]

## Innovation Flagged

**Innovation: Logical necessity as the mark-earning criterion in demonstrative encoding** — Rubric anchor: Visual Economy. In data-reporting visualization, the earning criterion for a mark is data representation. In demonstrative encoding, the earning criterion is logical necessity: a mark earns its place if removing it would invalidate the proof (or its visual presentation). This is a strictly stronger criterion than data-ink ratio — no redundancy is permitted that would not also be permitted in the logical argument. The rubric's Visual Economy dimension should recognize logical necessity as the relevant economy criterion for demonstrative encoding works.
