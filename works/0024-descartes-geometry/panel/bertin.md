# Bertin Review — La Géométrie Diagrams (1637)

## Visual Variable Audit (Demonstrative Mode)

As with Euclid (work 0018), I note that Descartes's diagrams operate under the demonstrative encoding contract: the visual variables are reassigned from their standard perceptual roles to geometric relational roles.

**Position:** Encodes geometric relationships, not geographic location. The relative position of points A, B, C defines the construction. This is the same reassignment as in Euclid — correct for demonstrative encoding.

**Size (segment length):** Here Descartes introduces something Euclid does not have: size as algebraic quantity. In Euclid, a segment's length is either equal to, greater than, or less than another — ordinal, not quantitative. Descartes's unit segment converts segment length from ordinal (relational) to quantitative (numerical). A segment of length 2 in Descartes's system is twice as long as the unit segment — a genuine quantitative ratio, not just a comparison.

This is a significant visual variable shift. In Euclidean demonstration, size is relational. In Cartesian demonstration, size is quantitative (relative to the unit). The encoding contract changes midway through the school's history.

Innovation #192: **Metric vs. ordinal segment encoding in mathematical diagrams** — in classical Euclidean diagrams, segment lengths are ordinal (one is equal to, greater than, or less than another, but no numerical value is assigned). In Cartesian analytic geometry diagrams, segment lengths are metric (numerical values are assigned via the unit segment). This is a change in the visual variable's perceptual property: from ordinal to quantitative. The diagrammatic contract must declare which mode is in effect.

**Labels:** In Euclid, labels identify geometric elements (this point is A, this segment is AB). In Descartes, labels also carry algebraic values: segment AB has length a, where a is an algebraic variable. The label does double duty — geometric identity and algebraic variable name. This is a dual encoding of labels not present in pure Euclidean diagrams.

Innovation #193: **Dual-function labels in hybrid geometric-algebraic diagrams** — labels in analytic geometry diagrams simultaneously name geometric elements (identity) and algebraic variables (quantity). The viewer must decode both simultaneously. This creates a compound decoding requirement not present in purely geometric or purely algebraic notation.

## Grammar Assessment

The Cartesian extension is internally consistent and builds faithfully on the Euclidean tradition. The key grammar additions:
1. Unit segment (declared) — converts ordinal lengths to metric lengths
2. Dual-function labels (undeclared as a system, but consistent in practice)
3. Reference line (the proto-coordinate axis) — undeclared but inferable from use

The undeclared additions (dual-function labels, reference line) create some Step E4 (diagrammatic framework declaration) gaps — the viewer must supply the convention that labels carry algebraic values. But the gaps are small: a mathematician reading Descartes in 1637 would have been familiar with algebraic variable notation from the contemporary algebraic literature.
