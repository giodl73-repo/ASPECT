# Bertin — 0026 The DEGAS Innovations Log as Visual Texture

## School Fidelity /20 → 16

I will conduct a visual variable audit of this work, because that is what I do, even in data art.

Variables in use:
- **Position (horizontal)**: work number (1–25). Ordered, quantitative. Correctly matched — horizontal position is the best ordered variable.
- **Position (vertical within column)**: ordinal innovation sequence within work. Ordered. Correctly matched.
- **Shape**: rubric dimension (6 mark types). Nominal. Correctly matched — shape is a selective, associative variable appropriate for nominal categories.
- **Color (hue)**: cluster family (warm / cool / neutral). Nominal/ordered hybrid. Partially correct — hue is nominally selective, but the warm-to-cool progression implies order, which is appropriate here since adopted→forming→unconfirmed is indeed an ordered progression.
- **Height of column**: innovation count per work. Quantitative. Correctly matched — height is an ordered, quantitative variable. This is the work's most efficient encoding.

Variable audit result: no grammatical errors. All variables are correctly matched to their data types. The color choice (warm=adopted, cool=forming) is the most interesting: hue is being used to encode both nominal (cluster identity) and ordered (cluster maturity) information simultaneously, which is a deliberate and defensible ambiguity.

Dock for the 3-per-row internal grid: this creates an artificial regularity that suggests a third variable (position within the 3-column internal grid) that does not encode anything. The eye will search for meaning in this regularity and find none. This is a grammar error of a specific type: phantom variable — a visual pattern that the eye decodes as an encoding but encodes nothing.

## Data/Story Integrity /20 → 15

The encoding is faithful to the underlying data. My concern is the summary of cluster membership into three color families (warm/cool/neutral) rather than 16 individual cluster colors. This is a necessary compression — 16 colors in a 6pt mark field would be illegible — but it sacrifices the individual cluster distinctions. The viewer cannot distinguish Cluster A from Cluster C from this image alone. This is appropriate for data art; it is a meaningful limitation.

## Legibility /15 → 10

The most legible encoding in this work is the column height (innovation count per work), which the eye reads as a skyline without instruction. The next most legible is the warm/cool color distribution, which conveys the adopted/forming cluster distinction as a global pattern. The mark type encoding (dimension by shape) requires the legend and close attention. Three levels of legibility, each requiring more viewer investment than the last. This is a reasonable design hierarchy for a gallery work.

## Visual Economy /15 → 11

The phantom variable (3-per-row internal grid) is the primary economy failure. Otherwise, the work is relatively economical for its school. The white ground is not wasted: it gives the columns visual separation and allows the field to breathe. The muted palette avoids the decorative excess that would make this feel like a dashboard rather than an art object.

## Resonance /15 → 12

The variable audit reveals that the work's primary resonance carrier is the column height distribution — the skyline of innovation density. This is the gestalt that carries before any mark is decoded. A viewer who spends 10 seconds at gallery distance sees: dense columns on the left, shorter columns in the middle, sparse columns on the right, with warm colors concentrated in the left-center region. This is a true and recognizable portrait of the DEGAS project's development. Type I resonance (Aesthetic-Intentional) is achieved at a solid level.

## Craft /15 → 12

The specification is precise. My execution concern: the phantom variable (3-per-row grid) will be very visible in a precisely executed rendering, which will make the grammar error more prominent. Fix this before rendering.

## My Total: 76/100

A competent data art work with one correctable grammar error (the internal column grid) and one inherent limitation (compression of cluster identities). The visual variable matching is sound; the column-height encoding is the strongest element; the mark-type encoding requires the legend but is available to the patient viewer.

## Innovation Flagged

**#206 — Phantom variable in texture-based data art**: when marks are arranged in a regular grid within a sub-region of a data art work, the eye decodes the regularity as an encoding even when it encodes nothing. This is a data art-specific grammar error not addressed in the current rubric. The rubric's Data/Story Integrity dimension (Step A: source quality declaration) does not cover this case. A new sub-step may be needed: "Is any visual regularity in the work's arrangement encoding something? If not, the regularity should be avoided or declared as layout-only."
