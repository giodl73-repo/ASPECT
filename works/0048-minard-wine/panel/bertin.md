# Bertin Review — 0048 Wine Export Flow Map

Variable assessment.

Band width: wine export volume. Size variable — quantitative, ordered. Correct.

Band position/path: geographic route from France to destination. Position — correct for geographic routing.

Band origin: all bands originate in France. This creates a visual convergence at France — the "hub" of a radial pattern. Radial flow patterns have a known limitation: when many bands originate from the same source, the source becomes crowded and individual bands can be difficult to distinguish near the origin. Minard handles this by using geographic routing (bands follow approximate routes to destinations) rather than direct lines, which spreads the bands along France's borders before radiating outward. This reduces the crowding at origin.

Band color: single color for all flows. No direction variable needed — all flows are unidirectional (France → destination). Compare to the Napoleon map, where two colors encode two directions (advance tan, retreat black). The wine map's commercial data is unidirectional, so single color is correct.

The multiple-band structure: when many bands radiate from a single source, the viewer must track which band goes to which destination. Labels at destinations are the solution. Minard uses destination labels correctly. The grammar is conventional; the execution is correct.

My concern specific to this map: the convergence of multiple bands near France's borders makes it difficult to compare bands of similar width — which is the largest export? The UK and Belgium bands are both substantial; if their widths are close, the viewer cannot confidently rank them without looking at the labels. The Napoleon map does not have this problem because there is only one band (plus retreat branches).

This is an inherent limitation of multi-source flow maps rather than a design failure, but it does create a small retinal efficiency cost.
