## Coercion Mechanism in Rubric Language

**Mechanism:** Process coercion with truthful encoding — the visualization honestly represents the output of a process that was itself designed to produce a specific political outcome. The map shows real district boundaries; the district boundaries were drawn to guarantee specific electoral results. The manipulation is invisible in the encoding.

**Distinguishing features:** (a) The visualization is fully accurate; (b) the coercion is in what is being shown, not how it is shown; (c) no alternative visualization in the same format could show the manipulation; (d) different visualization type (vote-share analysis, efficiency gap diagram) would be required to disclose the process.

**Rubric location:** This mechanism falls into a gap not covered by any current Integrity question. Question 3 (framing integrity) is the closest, but it addresses analytical framework disclosure, not process provenance disclosure. A v2.3 amendment should add a civic cartography modifier: when a map represents the output of a political process (district boundaries, zoning maps, administrative divisions), the process should be disclosed or the visualization should acknowledge its limitation.

**Purpose location:** 6.15/15. The legitimate civic reference function is served. The democratic deliberation function is not served. This is a partial Purpose success, not a failure — unlike the other Cluster X cases where purpose is manipulation. The gerrymandering case is a visualization that serves one civic purpose (reference) while failing another (deliberation).
